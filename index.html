¿Qué es ECMAScript?

ECMAScript no es un lenguaje de programación por sí mismo, sino un conjunto de normas que los lenguajes basados en él deben seguir. Fue creado por ECMA International, una organización que estandariza tecnologías para asegurar la compatibilidad entre diferentes productos y servicios. JavaScript es la implementación más conocida de ECMAScript, aunque también existen otros lenguajes que siguen su especificación, como JScript y TypeScript. La especificación de ECMAScript cubre temas como la sintaxis, tipos de datos, estructuras de control, funciones, y objetos básicos como arrays, strings y objetos. De esta forma, ECMAScript proporciona un marco común que asegura que los lenguajes basados en él compartan comportamientos similares, lo que permite a los desarrolladores escribir código más confiable y portable.

Versiones de ECMAScript

ES3 (1999)

ES3 fue la versión que introdujo características clave como el manejo de excepciones con `try/catch` y el soporte para expresiones regulares.

try {
  throw new Error("¡Error!");
} catch (e) {
  console.log(e.message); // ¡Error!
}

ES5 (2009)

ES5 introdujo el modo estricto (`"use strict"`) para ayudar a evitar errores comunes en el código. También añadió métodos como `forEach()` y `JSON`.

"use strict";
var nombres = ['Ana', 'Luis', 'Pedro'];
nombres.forEach(function(nombre) {
  console.log(nombre); // 'Ana', 'Luis', 'Pedro'
});

ES6 / ES2015

ES6 es una de las actualizaciones más grandes y trajo características como `let`, `const`, clases, promesas, módulos, y funciones de flecha.

const saludo = (nombre) => `Hola, ${nombre}`;
console.log(saludo('Juan')); // 'Hola, Juan'

ES7 / ES2016

ES7 introdujo dos características principales: el método `Array.prototype.includes()` y el operador exponencial (`**`).

console.log([1, 2, 3].includes(2)); // true
console.log(2 ** 3); // 8

ES8 / ES2017

ES8 introdujo las funciones `async/await`, que hacen más fácil trabajar con promesas, y los métodos `Object.values()` y `Object.entries()`.

async function obtenerDatos() {
  const respuesta = await fetch('https://api.example.com');
  const datos = await respuesta.json();
  console.log(datos);
}

ES9 / ES2018

ES9 permitió el uso del operador spread en objetos y la adición de `Promise.finally` y mejoras en las expresiones regulares.

const obj1 = { a: 1 };
const obj2 = { ...obj1, b: 2 };
console.log(obj2); // {a: 1, b: 2}

ES10 / ES2019

ES10 agregó métodos como `Array.prototype.flat()`, `trimStart()`, y `trimEnd()`, y también el método `Object.fromEntries()`.

const arr = [1, [2, [3]]];
console.log(arr.flat(2)); // [1, 2, 3]

const obj = { a: 1, b: 2 };
console.log(Object.fromEntries(Object.entries(obj))); // {a: 1, b: 2}

ES11 / ES2020

ES11 introdujo el encadenamiento opcional (`?.`), el operador de fusión nula (`??`), y el objeto global `globalThis`.

const usuario = { perfil: { nombre: 'Luis' } };
console.log(usuario.perfil?.nombre); // 'Luis'

let valor = null ?? 'valor predeterminado';
console.log(valor); // 'valor predeterminado'

ES12 / ES2021

ES12 presentó asignación lógica (`||=`, `&&=`, `??=`), además de `String.replaceAll()`.

let a = null;
a ??= 5;
console.log(a); // 5

console.log('Hola, mundo'.replaceAll('o', '0')); // 'H0la, mund0'

ES13 / ES2022

ES13 introdujo `top-level await`, la función `.at()` en arrays y un nuevo mecanismo para crear errores con `Error.cause`.

const datos = await fetch('https://api.example.com').then(r => r.json());
console.log(datos);

const letras = ['a', 'b', 'c'];
console.log(letras.at(-1)); // 'c'

ES14 / ES2023

ES14 añadió métodos como `Array.prototype.toSorted()`, `toReversed()`, y `toSpliced()` para mejorar el manejo de arrays.

const arr = [3, 1, 2];
console.log(arr.toSorted()); // [1, 2, 3]

console.log(arr.toReversed()); // [2, 1, 3]

¿Cómo utilizar ECMAScript?

  • 1. En navegadores: Escribe directamente tu código en archivos .js, ya que los navegadores modernos lo soportan de forma nativa.
  • 2. En servidores: Utiliza plataformas como Node.js, que también sigue la especificación ECMAScript.
  • 3. Compatibilidad: Para navegadores antiguos, puedes usar herramientas como Babel para convertir tu código a una versión más compatible.
  • 4. Con herramientas de desarrollo: Usa entornos como Webpack o Parcel para organizar y optimizar tu proyecto ECMAScript.
  • 5. Con módulos: ECMAScript permite dividir tu código en archivos reutilizables usando las palabras clave import y export.
  • 6. En frameworks: Al usar tecnologías como React, Vue.js o Angular, ya estarás aprovechando las características de ECMAScript.

Diferencias entre versiones

Cada versión de ECMAScript trae mejoras y nuevas características. Por ejemplo:

  • ES3 (1999): Estableció las bases de JavaScript, añadiendo manejo de errores con try/catch y mejorando los objetos estándar como Array y Date.
  • ES5 (2009): Introdujo el modo estricto, nuevos métodos para arrays como forEach y map, y funcionalidades mejoradas para objetos.
  • ES6 (2015): Aportó cambios radicales, como las declaraciones let y const, funciones flecha, clases, promesas, y soporte para módulos.
  • ES7 (2016): Añadió el operador de exponenciación (**) y el método Array.prototype.includes().
  • ES8 (2017): Introdujo async/await para manejar código asíncrono y nuevos métodos para objetos y cadenas.
  • ES9 (2018): Mejoró la manipulación de objetos con el operador de propagación (...) y añadió soporte para iteración asíncrona.
  • ES10 (2019): Añadió métodos como Array.prototype.flat() y String.prototype.trimStart().
  • ES11 (2020): Introdujo el encadenamiento opcional (?.), la fusión nula (??) y el tipo de datos BigInt.

Ejemplos de Funcionalidades por Versión

ECMAScript 3 (ES3)

ES3 consolidó el manejo de errores con try/catch.

// Manejo de errores con try/catch
    try {
      let x = undefinedVariable; // Error
    } catch (error) {
      console.error("Ha ocurrido un error:", error);
    }

ECMAScript 5 (ES5)

ES5 introdujo el "modo estricto" y métodos de arrays.

// Modo estricto
    "use strict"; // Habilita el modo estricto
    let x = 10;
    x = 20; // No hay problemas con la asignación
    
    // Métodos de arrays
    const arr = [1, 2, 3, 4];
    arr.forEach(num => console.log(num));  // 1, 2, 3, 4

ECMAScript 6 (ES6)

ES6 añadió let, const, clases, y promesas.

// Declaración de variables con let y const
    let x = 10;
    const y = 20;  // No se puede cambiar el valor de "y"
    
    // Funciones flecha
    const sum = (a, b) => a + b;
    console.log(sum(2, 3));  // 5
    
    // Clases
    class Person {
      constructor(name) {
        this.name = name;
      }
      greet() {
        console.log(`Hola, mi nombre es ${this.name}`);
      }
    }
    const person = new Person("Juan");
    person.greet();  // Hola, mi nombre es Juan
    
    // Promesas
    let promise = new Promise((resolve, reject) => {
      let success = true;
      if(success) {
        resolve("Operación exitosa");
      } else {
        reject("Error");
      }
    });
    promise.then(response => console.log(response)).catch(error => console.error(error));

ECMAScript 7 (ES7)

ES7 añadió el operador de exponenciación y el método includes() para arrays.

// Operador de exponenciación
    let result = 2 ** 3;  // 8
    
    // Método includes()
    const arr = [1, 2, 3, 4];
    console.log(arr.includes(3));  // true
    console.log(arr.includes(5));  // false

ECMAScript 8 (ES8)

ES8 introdujo async/await y métodos adicionales para objetos y cadenas.

// Async/Await
    async function fetchData() {
      let response = await fetch('https://api.example.com');
      let data = await response.json();
      console.log(data);
    }
    fetchData();
    
    // Nuevos métodos para objetos
    const obj = { name: 'John', age: 30 };
    console.log(Object.entries(obj));  // [["name", "John"], ["age", 30]]
    
    // Nuevos métodos para cadenas
    let str = "  Hola Mundo  ";
    console.log(str.trimStart());  // "Hola Mundo  "
    console.log(str.trimEnd());    // "  Hola Mundo"

ECMAScript 9 (ES9)

ES9 añadió la propagación de propiedades en objetos y la iteración asíncrona.

// Propagación de propiedades en objetos
    const obj1 = { a: 1, b: 2 };
    const obj2 = { ...obj1, c: 3 }; // Copia y agrega una nueva propiedad
    console.log(obj2);  // {a: 1, b: 2, c: 3}
    
    // Iteración asíncrona
    async function* generateAsync() {
      yield 1;
      yield 2;
      yield 3;
    }
    
    (async () => {
      for await (let value of generateAsync()) {
        console.log(value);  // 1, 2, 3
      }
    })();

ECMAScript 10 (ES10)

ES10 introdujo métodos flat() y flatMap() para arrays, y nuevos métodos para cadenas.

// Métodos flat y flatMap para arrays
    const arr = [1, [2, 3], [4, 5]];
    console.log(arr.flat());  // [1, 2, 3, 4, 5]
    
    const arr2 = [1, 2, 3];
    console.log(arr2.flatMap(x => [x, x * 2]));  // [1, 2, 2, 4, 3, 6]
    
    // Métodos de cadenas
    let str = "  Hola Mundo  ";
    console.log(str.trimStart());  // "Hola Mundo  "
    console.log(str.trimEnd());    // "  Hola Mundo"

ECMAScript 11 (ES11)

ES11 introdujo el encadenamiento opcional, la fusión nula y BigInt.

// Encadenamiento opcional
    const user = { profile: { name: "Alice" } };
    console.log(user.profile?.name);  // "Alice"
    console.log(user.profile?.age);   // undefined
    
    // Fusión nula (Nullish Coalescing)
    let foo = null;
    let bar = "Valor predeterminado";
    console.log(foo ?? bar);  // "Valor predeterminado"
    
    // BigInt
    const bigNumber = 1234567890123456789012345678901234567890n;
    console.log(bigNumber);  // 1234567890123456789012345678901234567890n
